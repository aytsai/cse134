Design Specs for Code 418

overview & requirements

Code 418's website will be a website that provides a QuizMaker and a QuizTaker where the user (Professor) will be able to create a quiz where his students will be able to take the quiz using our QuizTaker. The Client wants to save time in making quizzes for his classes and so he wants a website that will help save time by making it easier to make his quizzes. He also wants his students to have an easy time taking the quiz so they do not complain about having to go back to his typical printed quizzes. This means that Code 418's QuizMaker and QuizTaker will have to make creating the quizzes easy while also providing easy functionality for quiz taking.

audience

Professors...

are getting busier with their lives, and they want to be able to save time when they make quizzes for their students. Also, the rate of cheating has been going up during quiz taking so the professor might want to be able to have a functionality that will randomize the order of questions and answers for the quiz taker. We aim to create an intuitive and simple interface so that the professor spends as little time as possible adjusting to the new quiz making controls. Our website will provide everything they need in order for them to save time in making their quizzes.

Students...

have been complaining about the functionality of TED and other online test taking websites. Students have been wanting a new quiz taking website that will have them in mind and since we, the web designers, are also students, we're able to understand what they would want in a test taker.

functionality

QuizMaker

  1. Toolbar on side with choices of dragging types of questions onto droppable
  2. Once question type is selected options for the question comes up (Question is filled in first)
    1. Multiple Choice
      1. Will be able to add selectable answers
      2. Will be able to have multiple right answers by having check boxes next to the answer (Professor can select checkbox if its the correct answer)
    2. True/False
      1. Professor click on answer and will highlight in green for correct answer
    3. Fill in within a sentence
      1. Will be able to drag the fill-in around the sentence and also scale the width of the fill-in
    4. Fill-in after a sentence
    5. Short answer
      1. Scalable textbox
  3. Ability to delete answers or questions
  4. Ability to re-order the questions with drag/drop
  5. Ability to have randomized order of question/answer with click of a button

QuizTaker

  1. Navigation bar always on top of page for the questions.
    1. Show which questions are answered (Green)
    2. Show which questions are unanswered (Red)
    3. Each question have the option of selecting a checkable box that will let the user know they want to go back to that question in the navigation bar (Yellow)
  2. Be able to have option of showing only Red and Yellow questions in a group (Button on Navigational bar “Display all marked/unanswered questions” or “Display all unanswered questions”)
  3. Be able to swipe through questions for ease of use on tablets
  4. Have a up or down button next to question to jump to previous or next question
